Таблицы:

1. Таблица "User" — содержит информацию о клиентах:

Содержит поля:

    name: имя пользователя — тип поля строка (VARCHAR).
    surname: фамилия пользователя — тип поля строка (VARCHAR).
    birth_date: дата рождения пользователя — тип поля DATETIME.
    role: роль пользователя — тип поля строка (VARCHAR).
    email: электронная почта пользователя — тип поля строка (VARCHAR).
    password: пароль пользователя — тип поля строка (VARCHAR).
    login: логин пользователя — тип поля строка (VARCHAR).

Связи:

    Связана с таблицей "CustomerCardCredentials" связью "Один к одному" (у каждого клиента есть банковские учетные данные, и у каждой учетной банковской записи есть только один клиент).
    Связана с таблицей "Order" связью "Один ко многим" (у одного клиента может быть несколько заказов, но один заказ принадлежит только одному клиенту).
    Связана с таблицей "Logs" связью "Один ко многим" (один клиент может иметь множество логов, но каждый лог относится только к одному клиенту).
    Связана с таблицей "Favourite" связью "Один ко многим" (один клиент может иметь несколько избранных товаров, но каждый избранный товар принадлежит только одному клиенту).
    Связана с таблицей "Promocode" связью "Многие ко многим" (у одного клиента может быть несколько промокодов, один промокод может быть использован несколькими пользователями).

2. Таблица "Item" — содержит информацию о товарах:

Содержит поля:

    name: название товара — тип поля строка (VARCHAR).
    category_id: ID категории товара — внешний ключ на таблицу "ItemCategory".
    season: сезон товара — тип поля INT.
    sex: пол (целевая аудитория) товара — тип поля INT.
    price: цена товара — тип поля INT.
    size: размер товара — тип поля строка (VARCHAR).

Связи:

    Связана с таблицей "ItemCategory" связью "Один ко многим" (одна категория может включать несколько товаров, и один товар принадлежит одной категории).
    Связана с таблицей "Sale" связью "Один к одному" (один товар может иметь одну скидку).
    Связана с таблицей "Favourite" связью "Один ко многим" (один товар может быть добавлен в избранное несколькими клиентами).
    Связана с таблицей "Order" связью "Многие ко многим" (товар может быть во многих заказах и в заказе может быть много товаров).


3. Таблица "Order" — содержит информацию о заказах:

Содержит поля:

    total_amount: общая сумма заказа — тип поля INT.
    customer_id: ID клиента — внешний ключ на таблицу "User".
    order_date: дата заказа — тип поля DATETIME.
    delivery_id: ID доставки — внешний ключ на таблицу "DeliveryCredentials".
    item_id: ID товара — внешний ключ на таблицу "Item".

Связи:

    Связана с таблицей "User" связью "Один ко многим" (один клиент может иметь несколько заказов).
    Связана с таблицей "DeliveryCredentials" связью "Один к одному" (одному заказу соответствует одна доставка).
    Связана с таблицей "Item" через связь "Многие ко многим" (товар может быть во многих заказах и в заказе может быть много товаров).

4. Таблица "CustomerCardCredentials" — содержит учетные банковские данные клиентов:

Содержит поля:

    card_number: номер карты клиента — тип поля строка (VARCHAR).
    expire_date: срок действия карты — тип поля DATETIME.
    cvv_code: CVV код карты — тип поля строка (VARCHAR).

Связи:

    Связана с таблицей "User" связью "Один к одному" (каждому клиенту соответствует учетная запись с данными карты).

5. Таблица "Logs" — содержит информацию о действиях клиентов:

Содержит поля:

    customer_id: ID клиента — внешний ключ на таблицу "User".
    time: время события — тип поля DATETIME.
    type: тип события — тип поля строка (VARCHAR).
    text: текст сообщения — тип поля TEXT.

Связи:

    Связана с таблицей "User" связью "Один ко многим" (один клиент может иметь множество логов).

6. Таблица "ItemCategory" — содержит информацию о категориях товаров:

Содержит поля:

    name: название категории — тип поля строка (VARCHAR).
    is_new: новая ли категория — тип поля BOOLEAN.

Связи:

    Связана с таблицей "Item" связью "Один ко многим" (одна категория может включать несколько товаров).

7. Таблица "Promocode" — содержит информацию о промокодах:

Содержит поля:

    name: название промокода — тип поля строка (VARCHAR).
    percentage: процент скидки — тип поля INT.
    is_available: доступен ли промокод — тип поля BOOLEAN.
    time_from: дата начала действия — тип поля DATETIME.
    time_to: дата окончания действия — тип поля DATETIME.

Связи:

    Связана с таблицей "User" связью "Многие ко многим" (один промокод может быть использован несколькими клиентами и у клиента может быть много промокодов).

10. Таблица "Sale" — содержит информацию об акциях на товары:

Содержит поля:

    item_id: ID товара — внешний ключ на таблицу "Item".
    sale_price: акционная цена — тип поля INT.
    time_from: дата начала акции — тип поля DATETIME.
    time_to: дата окончания акции — тип поля DATETIME.

Связи:

    Связана с таблицей "Item" связью "Один к одному" (один товар идет одну скидку).

11. Таблица "Favourite" — содержит информацию об избранных товарах клиентов:

Содержит поля:

    customer_id: ID клиента — внешний ключ на таблицу "User".
    item_id: ID товара — внешний ключ на таблицу "Item".
    add_date: дата добавления в избранное — тип поля DATETIME.

Связи:

    Связана с таблицей "User" связью "Один ко многим" (один клиент может добавить в избранное несколько товаров).
    Связана с таблицей "Item" связью "Один ко многим" (один товар может быть добавлен в избранное несколькими клиентами).

12. Таблица "DeliveryCredentials" — содержит информацию о доставке:

Содержит поля:

    state: статус доставки — тип поля строка (VARCHAR).
    is_courier: является ли курьерской доставкой — тип поля BOOLEAN.
    address: адрес доставки — тип поля строка (VARCHAR).
    post_index: почтовый индекс — тип поля INT.
    cost: стоимость доставки — тип поля INT.

Связи:

    Связана с таблицей "Order" связью "Один к одному" (каждому заказу соответствует одна доставка).
